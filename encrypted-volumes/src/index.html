encrypted-volumes

rclone-crypt volumes over S3 - the bucket and the host only ever see ciphertext; keys enter here, over this deployment's in-enclave TLS
loading volumes…







  How this works. The App Config names where each volume's ciphertext lives (S3 endpoint, bucket, path) - never a key.
  Unlock with wallet asks your wallet to sign a fixed message naming the volume; the volume key is derived from that
  signature, so only the wallet holder can produce it - nothing is verified or stored on any server, and no transaction is made.
  Either way the key goes to this app only, which relays it over loopback to the in-enclave manager; the manager pulls the
  ciphertext and decrypts onto the CVM's encrypted ramdisk at /enc/<name>.
  Keys and plaintext never touch the operator's host or the bucket. Lock wipes the plaintext and drops the credentials;
  Push to cloud re-encrypts local edits back to the bucket (needs write-capable S3 credentials at unlock, and the volume
  not marked readOnly). A volume whose config carries a credsEnvelope (S3 credentials sealed under the same wallet
  signature) unlocks with one signature and nothing typed - after any restart, on any enclave.
