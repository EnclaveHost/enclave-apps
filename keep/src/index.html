Save writes plaintext to the enclave's in-memory ramdisk. Sync re-encrypts it and pushes ciphertext to your S3 bucket. Lock wipes the plaintext. The bucket and the host only ever see ciphertext.

keep keeps notes in a wallet-unlocked encrypted volume (rclone crypt over S3). The App Config names where the ciphertext lives — never a key. Unlock with wallet derives the volume key from a fixed signature; only the wallet holder can reproduce it, and nothing is verified or stored on any server. Once unlocked, /enc/<name> is an ordinary directory this app reads and writes with plain std::fs. Save persists to the enclave's ramdisk, Sync re-encrypts + pushes to your bucket, Lock wipes the plaintext. Verify the enclave before trusting it — enclave.host.
