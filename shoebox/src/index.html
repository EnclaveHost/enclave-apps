your files, decrypted only inside the enclave
Files decrypt into the enclave's in-memory ramdisk. Sync re-encrypts and pushes ciphertext to your S3 bucket; Lock wipes the plaintext. The bucket and the host only ever see ciphertext — the wallet that signs is the only key.
loading your lockers…
How this works. The App Config names where each volume's ciphertext lives (S3 endpoint, bucket, path) — never a key. Unlock with wallet asks your wallet to sign a fixed message naming the volume; the volume key is derived from that signature, so only the wallet holder can produce it — nothing is verified or stored on any server, and no transaction is made. The key goes to this app only, which relays it over loopback to the in-enclave manager; the manager pulls the ciphertext and decrypts onto the CVM's encrypted ramdisk at /enc/<name>. Uploads write plaintext there; Sync re-encrypts your changes back to the bucket; Lock wipes the plaintext and drops the credentials. Keys and plaintext never touch the operator's host or the bucket. Verify the code that holds your files with the deployment's remote attestation at enclave.host.
