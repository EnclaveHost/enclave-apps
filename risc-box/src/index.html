[ RISC Box ]
a real machine breathing on the enclave's own CPU — you just hold the console
This deployment requires an API key (the config's api_key).
This machine's CPU runs inside the enclave, the way a server-hosted emulator would — not in your browser. The enclave pulls the kernel and root filesystem from an S3 bucket, boots them on a vendored pure-Rust RISC-V system emulator, and streams the serial console here; your keystrokes go back into the guest. Save disk writes the guest's changes back to the bucket. The guest has a virtio-net NIC on an in-app user-mode network: the deployment's tcp:2222 port forwards to guest port 22, so an image with sshd accepts ssh -p 2222 straight into the machine.

How this works. RISC Box is an Enclave service app: one attested process holds the machine in the enclave's memory and serves this console over the deployment's in-enclave-terminated TLS. The machine is a vendored pure-Rust RISC-V system emulator (RV64GC, Sv39 MMU, CLINT/PLIC/UART/virtio-blk) compiled to the same wasm32-wasip2 target as the rest of the fleet and stepped under wasmtime in the TEE. Images are fetched from the configured S3 bucket over the platform's transparent egress (SigV4-signed when credentials are set, unsigned for public buckets). Credentials you type are sent only to this app, over TLS, and live only in enclave RAM. Verify the code running your machine with the deployment's remote attestation at enclave.host.
